# Ejemplos end-to-end

Cada ejemplo muestra cómo combinar varias SKILLs en un trabajo de auditoría real.

| Ejemplo | Tipo de auditoría | SKILLs involucradas |
|---|---|---|
| [`auditoria-ciberseguridad-nist/`](auditoria-ciberseguridad-nist/) | Madurez de ciberseguridad bajo NIST CSF 2.0 | `auditoria-ciberseguridad`, `planeacion-basada-riesgos`, `evaluacion-controles`, `papeles-trabajo`, `comunicacion-hallazgos` |
| [`auditoria-financiera-sox/`](auditoria-financiera-sox/) | Certificación SOX 404 sobre el ciclo de ingresos | `auditoria-financiera`, `auditoria-tecnologia-informacion`, `evaluacion-controles`, `muestreo`, `papeles-trabajo` |
| [`auditoria-esg-issb/`](auditoria-esg-issb/) | Aseguramiento limitado de divulgaciones IFRS S2 | `auditoria-esg-sostenibilidad`, `auditoria-ambiental`, `planeacion-basada-riesgos`, `evaluacion-controles`, `comunicacion-hallazgos` |

## Cómo leer un ejemplo

Cada carpeta contiene un único `README.md` con:

1. **Contexto del engagement** — qué empresa ficticia, qué alcance, qué restricciones.
2. **SKILLs activadas** — cuáles y por qué.
3. **Prompt inicial** — el mensaje real que un auditor le pasaría al agente.
4. **Output esperado** — qué debería producir el agente, paso a paso.
5. **Consideraciones** — riesgos, errores comunes, decisiones de juicio profesional.

> ⚠️ Los ejemplos usan **empresas y datos completamente ficticios**. Cualquier coincidencia con una organización real es involuntaria.
